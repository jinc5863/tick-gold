# 用户特别指定的重要规则

## 规则内容
**"你遇到任何问题必须先和我沟通！"**

## 具体含义
- 遇到任何困难、问题时，先与用户沟通
- 不能自行决定解决方案而不告知用户
- 必须让用户知道问题的存在和可能解决方向

## 应用范围
- 技术问题（构建失败、配置错误、路径问题等）
- 项目结构问题
- 设计决策
- 任何影响用户体验的操作

## 为什么要遵守
用户希望保持对项目进度的控制，确保问题得到正确理解和解决，避免不必要的返工和误解。

**为什么：** 用户强调沟通优先，以确保工作方式符合其预期和控制感。
